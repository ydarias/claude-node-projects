# Code Style

- Mappers are defined in their own file, using a static definition.
- Types are defined just once and reused instead of duplicating on each variable definition.
- Types and interfaces are defined in one file for each one and placed in a folder `src/types` for each module.
- File names use the kebab-case convention.
- It is preferred to use classes instead of plain functions to organize code.
    - Default is Object-Oriented Programming.
    - Ask before using a functional approach when it makes sense.
- Typescript sentences finish with a semicolon.
